# Объяснение доменной модели: BabayanRandomizer

Ниже — подробное объяснение каждого класса,
интерфейса и концепции, которая в нём используется.

---

## 🗂 Структура проекта ()

```
BabayanRandomizer/
├── Интерфейсы/
│   ├── IСписок.cs            ← Контракт для любого списка
│   └── IСлучайныйВыбор.cs   ← Контракт для сервиса рандомизации
├── Модели/
│   ├── Вариант.cs            ← Один вариант выбора ("Пицца")
│   ├── РезультатВыбора.cs    ← Результат одного случайного выбора
│   ├── СписокВариантов.cs    ← Список всех вариантов (агрегат)
│   └── ИсторияВыборов.cs    ← Журнал всех сделанных выборов
├── Сервисы/
│   └── РандомайзерСервис.cs  ← Логика: "как выбрать случайный вариант"
└── Program.cs                ← Точка входа, демонстрация работы
```

## Концепции ООП, использованные в проекте

### 1. Класс (Class)
**Что это:** Шаблон/чертёж для создания объектов.  
**Пример:** `Вариант` — это класс. Объект `new Вариант("Пицца")` — конкретный экземпляр.

### 2. Интерфейс (Interface)
**Что это:** Контракт без реализации. Указывает, ЧТО умеет делать класс, но не КАК.  
**Используется:** `IСписок<T>`, `IСлучайныйВыбор`.  
**Зачем:** Можно легко заменить реализацию (например, поменять `РандомайзерСервис`
на `КриптоРандомайзер`) без изменения остального кода.

### 3. Реализация интерфейса (`:`)
```csharp
public class СписокВариантов : IСписок<Вариант>  // реализует интерфейс
public class РандомайзерСервис : IСлучайныйВыбор    // реализует интерфейс
```
Класс после `:` **обязан** реализовать все методы и свойства интерфейса.

### 4. Агрегация (Aggregation)
**Что это:** Один объект СОДЕРЖИТ другие объекты, но они могут существовать отдельно.  
**Пример:**
- `СписокВариантов` содержит список `Вариант[]`.
- `ИсторияВыборов` содержит список `РезультатВыбора[]`.
- `РандомайзерСервис` получает `ИсторияВыборов` снаружи.

### 5. Инкапсуляция (Encapsulation)
**Что это:** Скрытие внутренних данных. Доступ только через методы/свойства.  
**Пример:** `_варианты` — приватное поле. Нельзя изменить напрямую снаружи.
Можно только через `Добавить()`, `УдалитьПоИндексу()` и т.д.

### 6. Обобщения (Generics) — `<T>`
**Что это:** «Тип-заглушка», подставляется при использовании.  
**Пример:** `IСписок<T>` → `IСписок<Вариант>` — список именно вариантов.  
**Зачем:** Один интерфейс работает с любым типом данных.

### 7. Перегрузка методов (Overloading)
**Что это:** Несколько методов с одним именем, но разными параметрами.  
```csharp
ВыбратьСлучайно(List<Вариант> варианты)
ВыбратьСлучайно(List<Вариант> варианты, List<Вариант> исключённые)
```

### 8. Переопределение `ToString()` (Override)
**Что это:** Изменение стандартного поведения метода, унаследованного от `object`.  
**Пример:** Вместо `BabayanRandomizer.Модели.Вариант` выводится `[ ] Пицца`.

## 📐 Диаграмма связей

```
IСписок<T> ←──── реализует ──── СписокВариантов
                                    │ содержит (агрегация)
                                    ▼
IСлучайныйВыбор ←─ реализует ─ РандомайзерСервис
                                    │ использует
                                    ▼
                             ИсторияВыборов ──── содержит ──→ РезультатВыбора
                                                                    │ ссылается на
                                                                    ▼
                                                                  Вариант
```

##  Пользовательские истории → Код

| История | Реализация |
|---|---|
| Вводить несколько вариантов | `СписокВариантов.Добавить(new Вариант("..."))` |
| Видеть результат на экране | `РезультатВыбора.ToString()` → вывод в консоль |
| Редактировать / удалять | `СписокВариантов.Редактировать()` / `УдалитьПоИндексу()` |
| Исключать уже выбранные | `Вариант.УжеВыбран`, `РандомайзерСервис.ВыбратьСлучайно(..., исключённые)` |
| Очистить список | `СписокВариантов.ОчиститьВсё()` |

## Ключевые слова 

| Слово | Значение |
|---|---|
| `public` | Видно всем |
| `private` | Видно только внутри класса |
| `readonly` | Нельзя переприсвоить после создания |
| `sealed` | Нельзя наследоваться |
| `override` | Переопределить метод родителя |
| `interface` | Контракт без реализации |
| `=>` | Краткая запись свойства/метода |
| `?` после типа | Тип может быть null |
| `var` | Тип определяется автоматически (вывод типа) |
